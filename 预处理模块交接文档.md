# 预处理模块交接文档

## 一、概述

预处理模块负责视频帧采集、人脸检测/追踪/活体检测、人脸注册匹配，是系统的数据入口。

**与接口模块的关系**：
- 接收指令：通过 `PreprocessingService.handle_command()` 处理界面模块下发的命令，由 `PreprocessingCommandAdapter` 适配为回调格式
- 输出数据：通过构造时注入的 `ui_callback` / `feature_callback` / `camera_list_callback` 向上推送

**与数据库模块的关系**：
- 不直接 import 数据库模块，通过 `UnifiedDataManager` 注入的回调解耦
- 写入路径：`set_face_embedding_writer()` 注入写回调 → `register_face` 处理中调用
- 读取路径：`load_faces_from_db()` 由 `UnifiedDataManager` 在启动时调用

**数据接口**（详见 `doc/接口文档.md`）：
- **PRI-01**：视频帧 + 人脸坐标 → 界面模块
- **PRI-02**：特征数据包（含 `face_matched`）→ 特征提取模块
- **PRI-03**：摄像头列表 → 界面模块
- **PRI-04**：人脸注册异步结果 → 界面模块（通过 `ui_callback`，`type: "face_registration_result"`）

---

## 二、命令接口

所有命令在 `PreprocessingService.handle_command()` 中实现，由 `PreprocessingCommandAdapter` 透传。

### 2.1 register_face 命令

| 项目 | 说明 |
|------|------|
| 指令名称 | `register_face` |
| 调用链路 | `FaceRegistrationDialog._on_complete()` → `registration_completed` 信号 → `MainWindow.on_face_registration_completed()` → `InterfaceManager.register_face()` → `_preprocessing_callback("register_face", ...)` |
| 触发时机 | 用户完成 4 姿势人脸采集后，在完成页选择存储方式时调用 |

**参数：**

```python
{
    "student_name": str,        # 学生姓名
    "frames": List[np.ndarray], # BGR 原始帧列表（4 关键帧 + 60 自动采样帧 ≈ 64 帧）
    "storage_type": str,        # "local" 或 "temp"
    "face_id": str,             # UI 生成的 UUID，"face_" 前缀 = local，"temp_" 前缀 = temp
}
```

**同步返回值（ACK）：**

```python
{
    "success": bool,
    "face_id": str,
    "msg": str  # "processing"
}
```

返回值应立即返回（如 `{"success": True, "face_id": face_id, "msg": "processing"}`），实际处理结果通过 PRI-04 异步通知。

**需实现的功能：**

1. **异步处理**（在 `handle_command` 中启线程或放入任务队列，不阻塞命令回调）：
   - 对每帧调用 InsightFace 检测 → 裁剪 → 提取 embedding（512d float32）
   - 收集所有有效 embedding

2. **内存注册表维护**：存入 `_face_registry` 字典：
   ```python
   _face_registry[face_id] = {
       "student_name": str,
       "embeddings": List[np.ndarray],  # 每帧一个 512d 向量
       "storage_type": "local" | "temp",
       "registered_at": float
   }
   ```

3. **持久化**（仅 `storage_type == "local"`）：
   - 调用注入的 `_face_embedding_writer` 回调落库
   - 回调签名：`Callable[[str, str, List[Tuple[bytes, str]]], bool]`
   - 参数：`(face_id, student_name, [(embedding_blob, pose_type), ...])`
   - `embedding_blob`：`np.ndarray.astype(np.float32).tobytes()`
   - `pose_type`：`"frontal"`, `"left"`, `"right"`, `"down"` 或 `"auto"`

4. **异步结果通知**（处理完成后，无论成功/失败）：
   ```python
   self.ui_callback({
       "type": "face_registration_result",
       "success": True,
       "face_id": face_id,
       "student_name": student_name,
       "msg": "注册成功",
       "embeddings_count": N,
   })
   ```
   此数据包由 `UnifiedDataManager._on_preprocessing_ui_packet()` 按 `type` 字段路由至 `InterfaceManager.on_face_registration_result()`。

### 2.2 query_face_registry 命令

| 项目 | 说明 |
|------|------|
| 指令名称 | `query_face_registry` |
| 调用链路 | `MainWindow.refresh_face_list()` → `UnifiedDataManager.generate_face_ids_with_details()` → `InterfaceManager.query_face_registry()` → `_preprocessing_callback("query_face_registry", {})` |
| 触发时机 | 程序启动、人脸注册成功后、切换到监控模式时 |

**参数：** 无

**返回值：**
```python
{
    "success": True,
    "faces": [
        {
            "face_id": str,
            "student_name": str,
            "storage_type": str,   # "local" 或 "temp"
            "registered_at": float,
        },
        ...
    ]
}
```

直接返回 `_face_registry` 中所有人的摘要（不含 embedding），用于界面展示可选人脸列表。

**注意**：若此命令返回失败或预处理模块未实现，`UnifiedDataManager.generate_face_ids_with_details()` 会自动降级到 `database_service.query_registered_faces()` 从数据库查询。预处理模块无需关心降级逻辑。

### 2.3 toggle_capture 命令（含 monitored_faces）

| 项目 | 说明 |
|------|------|
| 指令名称 | `toggle_capture` |
| 调用链路 | `MainWindow.on_start_analysis()` / `MainWindow.on_stop_analysis()` → `InterfaceManager.toggle_capture()` → `_preprocessing_callback("toggle_capture", ...)` |
| 触发时机 | 用户点击"开始分析"/"停止分析"按钮 |

**参数：**

```python
{
    "device_id": int,
    "start": bool,
    "monitored_faces": List[str],  # 要监控的 face_id 列表，可为空
}
```

- 当 `monitored_faces` 非空时：仅匹配列表中指定的 face_id，其余检测到的人脸忽略或标记为 "unknown"
- 当 `monitored_faces` 为空或未传时：匹配所有已注册人脸（保持现有行为）

## 三、初始化对接

预处理模块不直接 import 数据库模块，所有数据库交互通过 `UnifiedDataManager` 注入的回调完成。

### 3.1 写入路径：set_face_embedding_writer

`PreprocessingService` 需提供 `set_face_embedding_writer(callback)` 方法，供 `UnifiedDataManager.initialize_real_backend()` 注入。

- 回调签名：`Callable[[str, str, List[Tuple[bytes, str]]], bool]`
- 参数：`(face_id, student_name, [(embedding_blob: bytes, pose_type: str), ...])`
- 注入时机：`initialize_real_backend()` 中通过 `hasattr(service, "set_face_embedding_writer")` 检查后调用
- 使用时机：`register_face` 处理中 `storage_type == "local"` 时调用，`temp` 时不调用

### 3.2 读取路径：load_faces_from_db

`PreprocessingService` 需提供 `load_faces_from_db(faces_data)` 方法，供 `UnifiedDataManager.initialize_real_backend()` 在启动时调用，将数据库中的持久人脸载入内存注册表。

**调用流程**：
```
UnifiedDataManager.initialize_real_backend()
  → database_service.load_all_face_embeddings()          # 从 DB 读取
  → service.load_faces_from_db(faces_data)               # 载入预处理模块内存
```

**方法签名：**
```python
def load_faces_from_db(self, faces_data: List[Dict[str, Any]]):
    """程序启动时加载持久人脸数据到内存注册表
    
    Args:
        faces_data: [{"face_id": str, "student_name": str, "registered_at": float,
                       "embeddings": [{"embedding": bytes, "pose_type": str}, ...]}, ...]
    """
    for f in faces_data:
        self._face_registry[f["face_id"]] = {
            "student_name": f["student_name"],
            "embeddings": [np.frombuffer(e["embedding"], dtype=np.float32)
                           for e in f["embeddings"]],
            "storage_type": "local",
            "registered_at": f["registered_at"],
        }
```

## 四、匹配管线（帧处理循环）

预处理管线 `process()` 方法中，在检测人脸后应执行匹配。本节描述匹配流程与输出格式要求。

### 4.1 人脸匹配流程

1. 对每个检测到的人脸 ROI 运行 InsightFace embedding 提取
2. 将提取的 embedding 与 `_face_registry` 中所有条目的 embedding 做余弦相似度比较
3. 余弦相似度 > 阈值（建议 0.6）时，认定为匹配成功，赋予对应的 `face_id` 和 `student_name`
4. 未匹配的人脸分配临时 tracking ID，`student_name = "unknown"`

### 4.2 主人脸选择逻辑（由预处理负责人自行实现）

当画面中检测到多个人脸时，需选择主人脸：

1. **优先级 1**：匹配到 `monitored_faces` 中指定 face_id 的人脸（置信度最高的）
2. **优先级 2**：若未指定 monitored_faces 或无匹配，使用现有几何启发式规则（最大人脸、最靠近中心、tracking score 最高）
3. **优先级 3**：若只有一张未知人脸且无注册人脸，将其视为主人脸（face_id=tracking ID, student_name="unknown"）
4. **无人脸**：标记为 "主人脸缺失"（face_id=null, student_name="unknown"）

### 4.3 feature_faces 输出格式

匹配后的 `feature_faces` 每项应包含身份信息及比对结果：

```python
{
    "face_id": str,           # 匹配到的 face_id 或 tracking ID
    "student_name": str,      # 匹配到的 student_name 或 "unknown"
    "face_roi": np.ndarray,
    "confidence": float,      # 余弦相似度
    "face_matched": bool,     # 是否匹配到已注册人脸（余弦相似度 > 阈值，建议 0.6）
}
```

主人脸选择后，将主人脸的 `face_matched` 值填入 PRI-02 的 `FeatureFramePacket` 顶层字段，供特征提取模块做替学判断（参见 `doc/接口文档.md` PRI-02 和 FEI-01）。

## 五、数据包路由

UI 数据包（通过 `ui_callback` 发送）需统一加 `type` 字段以支持路由：

| type 值 | 用途 | 接收方 |
|---------|------|--------|
| `"face_registration_result"` | 人脸注册异步结果（PRI-04） | `InterfaceManager.on_face_registration_result()` |
| 无 type 字段 | 视频帧数据（PRI-01，向后兼容） | `InterfaceManager.on_video_frame_received()` |

路由逻辑在 `UnifiedDataManager._on_preprocessing_ui_packet()` 中实现。

---

## 六、注意事项

- 所有帧均为**原始 BGR numpy array**，不预裁剪。注册和匹配统一由 InsightFace 管线检测/裁剪，避免系统误差
- 临时人脸（`temp_` 前缀 face_id）仅在内存 `_face_registry` 中，应用退出时释放，不落库
- 人脸注册与人脸分析互斥，`register_face` 指令到达时保证摄像头已停止
- `_face_registry` 的读写需加 `threading.Lock` 保护（工作线程读写、命令线程写入）
- embedding 序列化：`np.ndarray.astype(np.float32).tobytes()`；反序列化：`np.frombuffer(blob, dtype=np.float32)`，维度固定 512
